Manual Testing Portfolio


Repozytorium zawiera moje praktyczne ćwiczenia z zakresu testowania manualnego, które wykonuję podczas tworzenia portfolio testera oprogramowania.


Zakres projektu

W tym repozytorium znajdują się:

✔️ Przypadki testowe (Test Cases)

Dokumenty przedstawiające krok po kroku scenariusze testowe dla wybranych funkcjonalności aplikacji webowych.

✔️ Checklisty

Listy kontrolne obejmujące m.in. testy UI, testy regresji oraz podstawową weryfikację elementów aplikacji.

✔️ Raporty błędów (Bug Reports)

Przykładowe zgłoszenia błędów sporządzone na podstawie znalezionych defektów wraz z opisem, krokami odtworzenia i oczekiwanym rezultatem.

✔️ Testy API (Postman)

Kolekcje zapytań API oraz dokumentacja testów wykonanych w Postmanie.

✔️ Testy eksploracyjne

Notatki z sesji testów eksploracyjnych przeprowadzonych na wybranych aplikacjach.


Aplikacje wykorzystane do nauki

Do ćwiczeń wykorzystuję ogólnodostępne aplikacje do nauki testowania:

Demoblaze – testy funkcjonalne i regresyjne (sklep internetowy)

SauceDemo – testy logowania, koszyka i przepływów użytkownika

Swagger Petstore API – testy API w Postmanie

The-Internet (Herokuapp) – zestaw różnych komponentów UI do analizy

Cel projektu

Celem tego portfolio jest:

rozwijanie praktycznych umiejętności testerskich,

udokumentowanie procesu nauki,

zaprezentowanie sposobu mojego myślenia i pracy,

przygotowanie się do pierwszej roli testera manualnego.


Status projektu

Repozytorium będzie regularnie aktualizowane, w miarę zdobywania nowych umiejętności oraz realizacji kolejnych zadań testerskich.
